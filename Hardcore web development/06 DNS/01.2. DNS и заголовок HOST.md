Пусть у нас есть хост, с полным доменным именем app.salesbeat.pro. На 80-м порту этого сервера поднят *NGINX*.

``` 
$ nc app.salesbeat.pro 80
HEAD / HTTP/1.1
HOST app.salesbeat.pro
...

HEAD / HTTP/1.1
HOST course.to.digital
...
``` 

*NGINX* обслуживает два *HTTP*-сервера, оба они работают на этом физическом хосте, но при этом находятся в разных доменах 
1. доменное имя первого сервера — app.salesbeat.pro
2. доменное имя второго сервера — course.to.digital

Система DNS резолвит оба этих доменных имени в один и тот же *IP*-адрес.

Используя заголовок запроса *HOST* — который, вообще говоря, является обязательным для протокола *HTTP* версии 1.1 и выше — *NGINX* понимает, какому из двух *HTTP*-серверов следует переадресовать полученный *HTTP*-запрос.

N.B.: все *HTTP*-запросы отправляются на *IP*-адрес некоторого сервера, и чаще всего ничего не знают о том, какое доменное имя у этого сервера.

Однако, начиная с версии 1.1 протокола *HTTP*, доменное сервера имя можно явно указать в заголовке *HOST* отправляемого *HTTP*-запроса.

В случае же, если один сервер обслуживает сразу несколько доменов, наличие заголовка *HOST* в запросе совершенно необходимо, и *NGINX* просто не будет обрабатывать *HTTP 1.1* запросы, не содержащие этого заголовка.
