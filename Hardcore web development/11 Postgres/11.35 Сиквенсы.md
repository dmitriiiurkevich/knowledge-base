*Сиквенс* — *sequence*, или *последовательность* — это объект схемы, который генерирует ряд уникальных целочисленных значений одного из следующих типов:
- serial
- smallserial
- bigserial
в порядке возрастания или убывания.

Сам по себе *сиквенс* не связан ни с каким *отношением* (ни с какой таблицей), но может быть использован для генерации значений тех или иных атрибутов некоторого *отношения*, чаще всего — суррогатного первичного ключа. Именно из-за отсутствия явной связи между *отношением* и используемым им *сиквенсом* значение суррогатного первичного ключа не уменьшается при удалении только что созданного экземпляра отношения.  

`\d` — возвращает список
- table
- view
- materialized view
- index
- sequence
- foreign table

`\d my_table` — описание таблицы — может включать описание *сиквенсов* 

```
rroom_db=> \d family

... 

Indexes:
    "family_pkey" PRIMARY KEY, btree (family_id)
Referenced by:
    TABLE "genus" CONSTRAINT "genus_family_id_fkey" FOREIGN KEY (family_id) REFERENCES family(family_id)
```

`\d my_sequence` — описание *сиквенса*

```
rroom_db=> \d family_family_id_seq
                    Sequence "public.family_family_id_seq"
  Type  | Start | Minimum |       Maximum       | Increment | Cycles? | Cache
--------+-------+---------+---------------------+-----------+---------+-------
 bigint |     1 |       1 | 9223372036854775807 |         1 | no      |     1
Sequence for identity column: public.family.family_id
```

Явное создание сиквенса 

``` sql
create sequence my_sequence
as bigint -- by default
start 1
minvalue 1 -- optional
maxvalue 9223372036854775807 -- optional
increment by 1 -- by default
cache 1 -- by default
no cycle -- by default
owned by none; -- by default
```

``` sql
select nextval('my_sequence');
select nextval('my_sequence');
select nextval('my_sequence');
```

``` sql
select currval('my_sequence');
```

``` sql
drop sequence my_sequence;
```


*Сиквенс*, может быть использован для генерации значений не только суррогатного первичного ключа, но других атрибутов отношения, например, для генерации уникальных целочисленных идентификаторов заказов.

``` sql
CREATE SEQUENCE orders_seq;
SELECT setval('orders_seq', (SELECT (TO_CHAR(NOW()::date, 'yyyymmdd') || '000')::bigint));
```

Важно замечание 1.
*Сиквенсы* работают вне транзакций. То есть *СУБД* гарантирует, что два пользователя одновременно вызвавшие *nextval* не получат одно и тоже число. А при откате транзакции значение *сиквенса* откачено не будет.

Важное замечание 2.
Поскольку вызовы `nextval` и `setval` не откатываются при откате транзакций, то *сиквенсы* не могут быть использованы для генерации непрерывных числовых последовательностей, если без исключительной блокировки на соответствующей таблице.

Пару слов о том как работает кэширование.

Если 
- значение параметра *cache* > 1, 
- *сиквенс* используется параллельно в нескольких сеансах, 
то последовательность сгенерированных значений может оказаться не вполне ожидаемой.

Каждый сеанс будет кэшировать несколько очередных значений *сиквенса*, и диапазон закэшированных значений будет обновляться при каждом вызове функции `nextval`. 

В результате, все значения *сиквенса*, уже закэшированные, но ещё не использованные в сеансе, будут потеряны при завершении этого сеанса, что приведёт к образовании
«дырок» в последовательности сгенерированных значений.

Более того, хотя разным сеансам гарантированно выделяются различные значения *сиквенса*, если рассмотреть все сеансы в целом, порядок этих значений может быть нарушен. 

Например, пусть у нас есть *сиквенс* со значением *cache* = 10 и два одновременно выполняемых сеанса: *A* и *B*.

Сеанс *A* может зарезервировать значения 1..10 и получить *nextval* = 1. 

Затем сеанс *B* может зарезервировать значения 11..20 и получить *nextval* = 11 ещё до того, как в сеансе *A* сгенерируется *nextval* = 2. 

Таким образом, при значении  *cache* = 1, можно быть уверенными в том, что *nextval* будет генерировать последовательные значения, но если *cache* > 1, то рассчитывать можно только на то, что все значения *nextval* будут различны, при этом их порядок может быть непоследовательным.

Функция `last_value` возвращает последнее зарезервированное значение для всех сеансов, вне зависимости от того, было ли оно уже возвращено функцией `nextval`.

А действие функции `setval` проявится в других сеансах только после того, как в них будут использованы все предварительно закэшированные значения.
