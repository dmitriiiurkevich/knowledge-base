
Терминальный мультиплексор позволяет одновременно работать с несколькими терминальными сессиями в одном окне:
- на сервере создаётся терминал с несколькими вкладками, 
- на локальной машине окно терминала делится на несколько частей, в каждой которых могут быть запущены любые приложения и процессы. 

При отключении от сервера текущая терминальная сессия будет сохранена.

## tmux

`$ sudo apt install -y tmux`

`$ tmux` — запустит *tmux*, создаст безымянную сессию с номером 0


`$ tmux attach -t 0`
`$ tmux attach -t my_session`
`$ tmux attach || tmux new` — возобновит работу *tmux* после отключения — запустит *tmux*, если есть сохранённая сессия, то присоединится к ней.

### Сессии

`$ tmux new-session -s my_session` — запустит *tmux*, создаст сессию с именем *my_session*
`$ tmux new-session -s my_session -d` — то же самое, но в фоновом режиме
  
| `Ctrl + b` | `"` | —   | запустит новую сессию |
| ---------- | --- | --- | --------------------- |

| `Ctrl + b` | `s` | —   | switch | —   | сменит сессию          |
| ---------- | --- | --- | ------ | --- | ---------------------- |
| `Ctrl + b` | `d` | —   | detach | —   | отсоединится от сессии |

`$ tmux ls` — вернёт список созданных сессий

`$ tmux kill-session -t 0`
`$ tmux kill-session -t my_session` — завершит сессию
`$ tmux kill-server` — завершит все сессии

### Окна

| `0 window` | `Ctrl + b` | `c` | —   | create | —   | создаст новое окно с индексом 0 |
| ---------- | ---------- | --- | --- | ------ | --- | ------------------------------- |
| `1 window` | `Ctrl + b` | `c` | —   | create | —   | создаст новое окно с индексом 1 |

| `Ctrl + b` | `0` | —   | перейдёт в окно с индексом 0 |
| ---------- | --- | --- | ---------------------------- |
| `Ctrl + b` | `1` | —   | перейдёт в окно с индексом 1 |

| `Ctrl + b` | `w` | —   | навигация по списку всех созданных окон |
| ---------- | --- | --- | --------------------------------------- |
| `Ctrl + b` | `q` | —   | отобразит номера всех открытых окон     |

| `Ctrl + b` | `%` | —   | разобьёт окно на две панели: правую и левую   |
| ---------- | --- | --- | --------------------------------------------- |
| `Ctrl + b` | `"` | —   | разобьёт окно на две панели: верхнюю и нижнюю |

| `Ctrl + b` | `←` | —   | перейдёт в левую панель окна   |
| ---------- | --- | --- | ------------------------------ |
| `Ctrl + b` | `→` | —   | перейдёт в правую панель окна  |
| `Ctrl + b` | `↑` | —   | перейдёт в верхнюю панель окна |
| `Ctrl + b` | `↓` | —   | перейдёт в нижнюю панель окна  |

### Настройки

`$ nano ~/.tmux.conf`
`$ sudo nano /etc/tmux.conf`

`set-option -g prefix C-q` — заменит комбинацию клавиш `Ctrl + b` на `Ctrl + q`.
`set -g mouse on` — позволит перемещать границы разделения окон курсора мыши

### Запуск сессий
Клиент-серверная арзитектура tmux позволяет полноценно управлять сессиями без непосредственного подключения к ним.

`$ tmux new-session -s my_session` 

`$ tmux split-window -v -t my_session:0.0` — разобьёт нулевую (дефолтную) панель нулевго окна сессии *my_session* вертикально
	`my_session` — указатель на сессию
	`my_session:0` — указатель на окно сессии
	`my_session:0.0` — указатель на панель окна

В результате выполнения команды в нулевом (дефолтном) окне `my_session:0` помимо нулевой (дефолтной) панели `my_session:0.0` появится новая панель `my_session:0.1`.

`$ tmux send-keys -t my_session:0.0 'my_command_1' Enter` 
`$ tmux send-keys -t my_session:0.0 'my_command_2' Enter` 
...

`$ tmux split-window -h -t my_session:0.1` — разобьёт первую панель нулевго окна сессии *my_session* горизонтально.

В результате выполнения команды в нулевом (дефолтном) окне `my_session:0` появится новая панель `my_session:0.2`.

`$ tmux send-keys -t my_session:0.2 'my_command_3' Enter`
`$ tmux send-keys -t my_session:0.2 'my_command_4' Enter` 
...

И так далее. 

Для всей этой длинной последовательности команд можно создать *alias*.
