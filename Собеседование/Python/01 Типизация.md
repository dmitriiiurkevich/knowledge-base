1\.
Статическая типизация — *static typing* — на этапе компиляции:
- *устанавливаются* конечные типы переменных и параметров функций,
- происходят проверки на соответствие типов.

Что, однако, не мешает скомпилированным программам ловить *kernel panic* на этапе выполнения.

Динамическая типизация — *dynamic typing* — на этапе выполнения:
- *выясняются* конечные типы переменных и параметров функций,
- происходят проверки на соответствие типов.

Опциональная статическая типизация — *optional static typing* — сочетает оба подхода.

2\.
Сильная типизация — *strong typing* — автоматические неявные преобразования типов по возможности не выполняются. Строку на число умножить можно, но строку с числом сложить нельзя. 

Слабая типизация — *weak typing* — часто выполняются автоматические неявные преобразования типов, для того, чтобы, по возможности, избежать ошибок на этапе выполнения. Даже если в результате таких преобразований может происходить потеря точности или логика таких преобразований не всегда очевидна.

3\.
Явная типизация — *manual typing* — конечные типы должны быть явно заданы пользователем.

Неявная типизация — *type inference* — конечные типы определяет компилятор или *type checker*.

Пример нерабочего кода:

```python 
def f():
	return "foo"

def g():
	return f() + 123

# mypy: ok 
# pytype: line 4 in g: unsupported operand type(s)
```


Проблема в том, что *mypy* не проверяет нетипизированные функции. Для решения этой проблемы можно использовать *явную типизацию*. А можно сменить *type checker* на более продвинутый, например, на *pytype*.

```python 
def f() -> str:
	return "foo"

def g() -> int:
	return f() + 123

# mypy: error: Unsupported operand types 
# pytype: line 4 in g: unsupported operand type(s)
```


Python — язык с сильной опциональной статической типизацией.
JS — язык со слабой динамической типизацией.
