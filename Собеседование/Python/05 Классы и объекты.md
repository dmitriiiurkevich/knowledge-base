

Существуют три базовые C-Python структуры:
1. PyObject — создаётся для любых объектов
2. PyVarObject — создаётся для объектов переменной длины, содержит в себе PyObject 
3. PyTypeObject — создаётся для объектов классов, содержит в себе PyVarObject


*N.B.*
Таким образом, каждый класс является объектом переменной длины.


```
┌─────────────────────┐ 
│ ob_refcnt           │ ← PyObject часть 
│ ob_type             │ 
│ ob_refcnt           │
│ dict (optional)     │ 
│ ...                 │ 
├─────────────────────┤ 
│ ob_size             │ ← PyVarObject часть 
│ ...                 │ 
├─────────────────────┤
│ tp_name             │ ← PyTypeObject часть 
│ tp_bases            │ 
│ tp_mro              │ 
│ tp_basicsize        │ 
│ tp_new              │
│ tp_init             │
│ tp_dict             │
│ tp_methods          │
│ tp_getset           │
│ ...                 │ 
└─────────────────────┘
```

Структура `PyObject` работает на уровне Python.

Структура `PyObject` — это базовая структура объекта, соответствующая структуре объекта экземпляра базового класса object.

Структура `PyObject` объектов пользовательских классов может быть расширена в соответствии с особенностями каждого конкретного класса. Например, применение в классе механизма `__slots__` меняет базовую структуру `PyObject` объектов этого класса.

Структура `PyTypeObject` работает на уровне C.

Структура `PyTypeObject` не может быть расширена новыми полями, её размер фиксирован.

Допускается только модификация следующих полей:
- `tp_methods` — можно добавлять через C API новые статически определённые C-методы — такие как системный метод `lower`, предопределённый в классе строк
 - `tp_getset` — можно добавлять C API новые C-свойства — такие как системное свойство `im`, предопределённое в классе комплексных чисел
- `tp_dict` — можно **добавлять любые атрибуты класса**, включая Python-методы, свойства, переменные и т.д.


В Python всё является объектом. Рассмотрим подробно, как устроены
1. структура объекта
2. структура объекта-класса
3. структура объекта-метакласса

### 1\. Структура объекта

Базовая структура объекта:
- `PyObject.dict` → опциональный словарь динамических атрибутов объекта
- `PyObject.ob_type` → ссылка на класс объекта
- `PyObject.ob_refcnt` → счётчик ссылок на объект
- ...

*N.B.*
Словарь атрибутов объекта содержится в `PyObject.dict`.

Словарь атрибутов класса объекта содержится в аналогичном словаре объекта-класса, доступного по ссылке из `PyObject.ob_type`.

### 2\. Структура объекта-класса

``` python
class MyClass: 
	pass
```

```
┌─────────────────────┐
│ ob_refcnt           │ ← PyObject часть 
│ ob_type → type      │ 
├─────────────────────┤ 
│ tp_name = "MyClass" │ ← PyTypeObject часть 
│ tp_new = ...        │ 
│ tp_init = ...       │ 
└─────────────────────┘
```


Структура объекта-класса отличается от структуры обычного объекта тем, что содержит не только `PyObject` часть, но и `PyTypeObject` часть.


|                        | В структуре обычного объекта | В структуре объекта-класса                          |
| ---------------------- | ---------------------------- | --------------------------------------------------- |
| `PyObject.ob_type`     | ссылка на класс объекта      | ссылка на метакласс объекта-класса → `type`<br>     |
| `PyTypeObject.tp_name` | —                            | C строка с именем этого объекта-класса  = `MyClass` |
|                        |                              |                                                     |

|                        | В структуре обычного объекта              | В структуре объекта-класса                                      |
| ---------------------- | ----------------------------------------- | --------------------------------------------------------------- |
| `PyObject.dict`        | словарь динамических атрибутов экземпляра | словарь атрибутов класса                                        |
| `PyTypeObject.tp_dict` | —                                         | ссылка на тот же самый словарь атрибутов класса, но на уровне C |


``` python
class MyMeta(type): 
	pass

class MyClass(MyMeta): 
	pass
```


|                        | В структуре обычного объекта | В структуре объекта-класса                          |
| ---------------------- | ---------------------------- | --------------------------------------------------- |
| `PyObject.ob_type`     | ссылка на класс объекта      | ссылка на метакласс объекта-класса → `MyMeta`<br>   |
| `PyTypeObject.tp_name` | —                            | C строка с именем этого объекта-класса  = `MyClass` |


### 3\. Структура объекта-метакласса


``` python
class MyClass(type): 
	pass
```

Метакласс — это или `type`, или подкласс `type`.

|                         | В структуре объекта-класса | В структуре объекта-метакласса |
| ----------------------- | -------------------------- | ------------------------------ |
| `PyObject.type`         | `type`                     | `type`                         |
| `PyTypeObject.tp_bases` | `(object,)`                | `(type,)`                      |
| `PyTypeObject.tp_mro`   | `(MyClass, object)`        | `(MyClass, type, object)`      |
| `PyTypeObject.tp_new`   | `object.__new__`           | `type.__new__`                 |


``` python
class MyMeta(type):
	pass

class MyClass(MyMeta): 
	pass
```


|                         | В структуре объекта-класса | В структуре объекта-метакласса    |
| ----------------------- | -------------------------- | --------------------------------- |
| `PyObject.type`         | `type`                     | `MyMeta`                          |
| `PyTypeObject.tp_bases` | `(object,)`                | `(MyMeta, type,)`                 |
| `PyTypeObject.tp_mro`   | `(MyClass, object)`        | `(MyMeta, MyClass, type, object)` |
| `PyTypeObject.tp_new`   | `object.__new__`           | `type.__new__`                    |

### Структуры классов object и type

Класс `object` — универсальный класс, общий предок всех классов.
Класс `type` — универсальный метакласс, общий предок всех метаклассов.


```
PyTypeObject для object: 
┌─────────────────────────┐ 
│ ob_type → type          │ ← object создан метаклассом type 
│ tp_name = "object"      │ 
│ tp_bases = ()           │ ← object не имеет предков
│ tp_mro = (object,)      │ 
└─────────────────────────┘ 

PyTypeObject для type: 
┌─────────────────────────┐ 
│ ob_type → type          │ ← type создан самим собой 
│ tp_name = "type"        │ 
│ tp_bases = (object,)    │ ← type наследуется от object! 
│ tp_mro = (type, object) │ 
└─────────────────────────┘
```


### Порядок создания объекта

`PyTypeObject.tp_new` → ссылка на метод `__new__` класса
`PyTypeObject.tp_init` → ссылкf на метод `__init__` класса

#### Коротко:
- Python вызывает `__new__` класса
- Метод `__new__` создаёт и возвращает экземпляр
- Если `__new__` вернул экземпляр того же класса, Python автоматически вызывает `__init__`
- Метод `__init__` инициализирует экземпляр

#### Подробнее

Метод `__new__` 
- создаёт структуру PyObject для нового объекта
	- **Malloc**: Выделяется сырая память
	- **Memset**: Память обнуляется
	- **Type pointer**: Устанавливается указатель на тип
	- **Reference count**: Устанавливается в 1
	- **Weak references**: Обнуляется список слабых ссылок
	- **Dictionary**: При необходимости создаётся место под `PyObject.dict` (но не сам словарь)
- возвращает ссылку на только что созданный объект. 

Метод `__init__`
- через переменную `self` забирает из метода `__new__` только что созданный объект
- инициализирует объект
	- при необходимости создаёт словарь `PyObject.dict`
	- сохраняет в только что созданном словаре `PyObject.dict` динамические атрибуты объекта и ссылки на их значения

Метод `__init__` не будет вызван, если метод `__new__` вернёт объект не своего класса.


*N.B.*
Если создаваемый объект, является объектом класса, то перед вызовом метода `__new__` будет запущен метод `__call__` соответствующего метакласса. 

*N.B.*
Инициализация объектов системных immutable классов (и классов, расширяющих их) происходит с помощью метода `__new__`. Метод в системных immutable классах ничего не делает.
