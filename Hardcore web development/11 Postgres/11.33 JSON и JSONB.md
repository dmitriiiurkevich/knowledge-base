Да, индекс указывает в плане и реальности на всего одну запись и достать одну запись легко. Но в то же время надо понимать, что подход «просто накинуть индекс» работает не всегда — как минимум зависит от селективности индекса, то есть характеристики данных, которая к тому же может со временем меняться, если данные не статичны, обновляются-дополняются. То есть, глядя на create table и select из этой таблицы, не всегда можно гарантировать, что индекс окажет влияние на запрос и будет использован.

Например, если нам надо достать книгу не по названию, а по количеству страниц, то индекс может работать не так предсказуемо:

копировать код

```sql
create index on book(pages);
create index on book_json using btree (((data->>'pages')::int));

vacuum full analyze book, book_json;

explain analyze
select * from book where pages between 1000 and 1200;
-- cost up to 11330
-- индекс используемся

explain analyze
select * from book_json where (data->'pages')::int between 1000 and 1200;
-- cost up to 26202
-- индекс не используется, идет сканирование всей таблицы
```

Если изменить условия выборки, уменьшив селективность, то и в случае `book` индекс может перестать использоваться:

копировать код

```sql
explain (analyze, buffers)
select * from book where pages between 1000 and 2600;
-- cost up to 24346
-- индекс не используется
```
