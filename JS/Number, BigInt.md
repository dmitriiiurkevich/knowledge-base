
Тип данных `number` 
— число сплавающей точкой и двойнйой точночтью (IEEE 754)
— полный аналог `double` в Java. 

Долгое время был единственным числовым типом в JavaScript.

|       | sign |     | mantissa |     | exponent |     |     |
| ----- | ---- | --- | -------- | --- | -------- | --- | --- |
| bits: | 1    | +   | 52       | +   | 11       | =   | 64  |

```
number_value == (-1)^sign × (1 + mantissa) × 2^(exponent - 1023)
```

Стандартная проблема `int` в Java — переполнение.

```
JavaScript: 2147483647 + 1 ==  2147483648
Java:       2147483647 + 1 == -2147483648
```

Использование стандарта IEEE 754 решает проблему переполнения, но приводит к новой проблеме — потери точности даже при целочисленных вычислениях

```
JavaScript: 9007199254740992 + 1 == 9007199254740992
JavaScript: 9007199254740992 + 2 == 9007199254740994
```




Для решения, в частности, именно этой проблемы в 2020-м году в JavaScript (ES2020) появился новый тип данных — `BigInt`.
