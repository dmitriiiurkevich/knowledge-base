Компилируемые языки — *compiled languages*: *C*, *C++*, *Go*, *Rust* — преобразуют программу в исполняемый файл с помощью компилятора, после чего, для запуска программы на исполнение, необходимо будет запустить этот самый исполняемый файл.

После запуска исполняемого файла операционная система создаст новый процесс, в *text segment* виртуальной памяти которого, будет загружен код исполняемого файла.

Интерпретируемые языки — *interpreted languages*: *Python*, *JS*, *Java*, *Lua* — не создают самостоятельный исполняемый файл.

Поэтому для запуска программы на исполнение, из командной строки запускается виртуальная машина, содержащая интерпретатор соответствующего языка, 

И операционная система создаёт процесс, в *text segment* виртуальной памяти которого, загружается код виртуальной машины, содержащий интерпретатор. 

Далее происходит одно из двух.

Первый вариант:
Один из параметров команды, запускающей процесс виртуальной машины, содержит путь к файлу, содержащему текст запускаемой программы.

Этот текст последовательно считывается и преобразуется в байткод  — наборы инструкций внутреннего языка виртуальной машины — которые последовательно загружаются в heap процесса запущенной виртуальной машины

Второй вариант:
Один из параметров команды, запускающей процесс виртуальной машины, содержит путь к файлу, содержащему не текст запускаемой программы, а уже готовый байткод, предварительно сгенерированный на основе текста программы.

В этом случае байткод загружается напрямую в heap процесса запущенной виртуальной машины.
