Как мы уже отмечали выше, каждый процесс считает, что для его виртуального адресного пространства может быть выделена вся физическая память системы, то есть:

| Шина данных | Шина адреса | Объём           |
| ----------- | ----------- | --------------- |
| 32 bit      | 32 bit      | 2³² bit = 4 GB  |
| 64 bit      | 48 bit      | 2⁴⁸ bit = 35 TB |

Кроме того, в операционной системе реализован механизм отображения виртуальной памяти процесса на более медленную дисковую память — механизм *своппирования*.

Выделяемая процессу память делится на две неравные части:
- Kernel memory
	- process id — *PID*
	- process execution state
		- running, 
		- stopped, 
		- interruptible sleep, 
		- uninterruptible sleep, 
		- zombie
	- ...
	- link 
	- Memory management structures
		- page tables
		- memory maps
	- ...
	- File descriptor table
	- Scheduling information
		- priority
		- timeslices
	- ...
	- Security context
		- *UID*
		- *GID*
		- capabilities
	- ...
	- 
- Process memory — virtual memory
	- Text section — r-x
	- Data section
		- read only data — r--
		- read write data — rw-
	- Stack
	- Heap
