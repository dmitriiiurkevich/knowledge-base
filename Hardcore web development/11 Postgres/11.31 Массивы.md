
Массивы предназначены для хранения данных одного типа.

Использование массивов формально нарушает *1NF* принцип нормализации. Когда использование массивов оправдано?

Arrays can be used to denormalize data and avoid *lookup tables*. A good rule of thumb for using them that way is that you mostly use the array as a whole, even if you might at times search for elements in the array. Any array processing, that is heavier than simple search, is going to be more complex than a *lookup table*.

Многомерные координаты точки могут храниться в массиве, не слишком нарушая принципы нормализации.

``` sql
drop table if exists some_entity;

create table some_entity (
    `entity_id bigint generated always as identity primary key,`
    `int_array INT[],`
    `text_array TEXT[],`
    `date_array DATE[]`
);
```

``` sql
insert into some_entity (int_array, text_array, date_array) values (`
    `'{100, 255, 328}',`
    `'{"John", "Jack", "Mary"}',`
    `'{"2024-06-25", "2024-06-26", "2024-01-01"}'`
);

select * from some_entity;
```

``` sql
-- get element by index
select int_array[1], text_array[1], date_array[1] from some_entity;
--> 100	"John" "2024-06-25"
```

Размер массива можно ограничить

``` sql
drop table if exists some_entity;

create temp table some_entity (
    entity_id bigint generated always as identity primary key,
    int_array INT[2],
    text_array TEXT[2],
    date_array DATE[2]
);
```

Но это не помешает положить в любой из этих массивов любое количество элементов.

``` sql
-- insert into array
update some_entity set int_array = array_append(int_array, 777);
select int_array from some_entity;

-- len of array
select array_length(int_array, 1) from some_entity;
-- here is 1 is a dimention of the array

-- search of array element
select int_array from some_entity where 777=any(int_array);
--> {100,255,328,777}
select int_array from some_entity where 12345=any(int_array);
--> no results
```

Array to string

``` sql
select '{10, 20, 30}'::integer[];

select array_to_string('{10, 20, 30}'::integer[], ', ');
select 'Array: ' || array_to_string('{10, 20, 30}'::integer[], ', ');
```

String to array

``` sql
select string_to_array('100, 200, 333', ', ');
--> 100, 200, 333
```

Функция *unnest* — array to list of strings

``` sql
select '{John, Jack, Mary}'::text[];
--> {John,Jack,Mary}

select unnest('{John, Jack, Mary}'::text[]);
--> virtual table with only a column 'unnest', containing 3 rows: 
--> John
--> Jack
--> Mary

select unnest(string_to_array('John, Jack, Mary', ', '));
--> virtual table with only a column 'unnest', containing 3 rows: 
--> John
--> Jack
--> Mary

select unnest(string_to_array('John, Jack, Mary', ', ')) as t order by t;
--> virtual table with only a column 't', containing 3 rows: 
--> Jack
--> John
--> Mary
```

Также *unnest* может принимать на вход несколько массивов

``` sql
SELECT 123 AS some_id, *
FROM unnest('{100, 40.5, 76}'::numeric[]
          , '{01-2222-05,01-3333-04,01-4444-08}'::text[]) AS u(amount, invoice);
--> virtual table with 3 columns: 'some_id', 'amount', 'invoice'
--> and containing 3 rows: 
--> 123, 100, 01-2222-05
--> 123, 40.5, 01-3333-04
--> 123, 76, 01-4444-08
```          

Многомерные массивы

``` sql
drop table if exists some_entity

create temp table some_entity (
    entity_id bigint generated always as identity primary key,
    text_array INT[][][][][]
);

insert into some_entity (text_array) VALUES (
    '{{{{{1, 2, 3, 4}}}}}'
);

insert into some_entity (text_array) VALUES (
    '{{{{{1, 2, 3}}}}}'
);

select * from some_entity;
--> {{{{{1, 2, 3, 4}}}}}
--> {{{{{1, 2, 3}}}}}
```

Решение задачи:

```sql
select
    log_id,
    created_at,
    (string_to_array(messages, '|'))[array_length(string_to_array(messages, '|'), 1)] as last_message
from log;
```

Здесь мы вызывает функцию `string_to_array`, передавая ей поле `messages` и символ, по которому мы будем разбивать это поле, символ `|`. На выходе функция отдаёт массив строк, от которого нам надо взять последний элемент. Чтобы рассчитать индекс этого последнего элемента, мы вычисляем длину массива.

Или:

копировать код

```sql
select
    log_id,
    created_at,
    split_part(messages, '|', array_length(string_to_array(messages, '|'), 1)) as last_message
from log;
```

Здесь мы используем функцию `split_part`, которая принимает на вход разбиваемую строку, разделитель и индекс нужного элемента полученного массива.

Или ещё проще (`split_part` умеет принимать отрицательные индексы — они отсчитываются от конца):

копировать код

```sql
select
    log_id,
    created_at,
    split_part(messages, '|', -1) as last_message
from log;
```
