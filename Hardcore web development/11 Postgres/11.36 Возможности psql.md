`\q`— выход

`\l` — список баз данных в текущем в кластере
`\l+` — расширенный вариант того же списка

`\du` — список ролей в текущем кластере, кроме системных
`\duS` — список всех ролей в текущем кластере, включая системные


`\dn` — список схем (*namespaces*) текущей базы данных, кроме системных
`\dn+` — расширенный вариант того же списка

`\dnS` — список всех схем (*namespaces*) текущей базы данных, включая системные
`\dnS+` — расширенный вариант того же списка

`\d` — список объектов схемы
- table
- view
- materialized view
- index
- sequence
- foreign table


`\dt` — список таблиц
`\ds` — список *сиквенсов*
...

`\d my_table` — описание таблицы
`\d my_sequence` — описание *сиквенса*
...

`\l` — список баз данных, развёрнутых в кластере
`\l+` — расширенный вариант того же списка


`\copy (SELECT * FROM author) TO '~/authors.csv' WITH CSV HEADER` — сохраняет результат запроса в *csv* файл 

`\o ~/my_file.txt` — перенаправляет вывод psql в файл
`\o` — перенаправляет вывод psql обратно в терминал

Можно даже выводить данные в файл в формате *HTML*

```
\html
\o ~/my_file.html

select * from author;

\o
\html
```

Длинные запросы удобно редактировать во внешнем текстовом редакторе. Команда `\e` запускает дефолтный редактор и открывает в нём временный файл. При сохранении текста запроса и выходе из редактора запрос выполнится.

```
\e

select * from author;
```

Справка о командах и *SQL* операторах

```
\h select

\h delete

\h update

\h create
\h create table
```


`\x` — *expanded display mode* — для отношений с большим количеством атрибутов

`\s ~/my_file.sql` — сохранение истории команд текущей сессии в файл.
