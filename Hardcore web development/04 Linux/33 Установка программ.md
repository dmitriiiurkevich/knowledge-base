
Есть три способа установки программ в Linux:
1. установка из пакетного менеджера
2. установка из исходников
3. установка из специальных пакетов
	- appimage
	- flatpak
	- snap

`$ sudo apt install -y my_packege` — скачивает пакет — уже скомпилированную программу.

## В чём преимущкство использования пакетного менеджера? 

У каждого пакета скачиваемого пакетным менеджером, есть свои зависимости. Какие-то из этих зависимостей могут быть уже быть установлеными в опеерационную систему, какие-то — нет. Все недостающие зависимости также ищутся в репозиториях пакетного менеджера.

### Возможен ли при этом *dependency hell*?

Как правило, — нет.   В момент релиза дистрибутива версии всех программ, доступных для скачивания в официальных репозиториях, фиксируются. Обновление, как правило, происходит только при следующем релизе дистрибутива.  Разработчик дистрибутива перед релизом должен проверить, что программы, доступные для скачивания в официальных репозиториях, не конфликтуют друг с другом.

`$ sudo apt remove my_package` — удаляет пакет.
`$ sudo apt purge my_package` — удаляет пакет вместе с конфигурационными файлами.

`$ sudo apt upgrade my_package` — обновляет пакет.
`$ sudo apt upgrade` — обновляет все установленные пакеты.

`$ apt search my_pattern | less` — ищет пакеты.

`$ apt show my_package | less` — отображает информацию о пакете.


## В каких случаях возникает необходимость установки из исходников? 

1.
В случае, если нам необходимо скомпилировать устанавливаемую программу некоторым специальным образом, — не таким, каким эта программа была скомпилирована в пакетном менеджере. 

2.
В случае, если мы хотим контролировать расположение на диске всех файлов, необходимых для выполнения устанавливаемой программы. 

К сожалению, apt при установке пакета может по своему усмотрению раскидать разные файлы, входящие в состав этого пакета, по разным папкам: сюда программу, сюда библиотеки, сюда конфиги и так далее. И иногда это бывает не очень удобно. 

### Возможен ли при этом *dependency hell*?

Ещё как возможен.

Технологии, созданные для решения проблемы *dependency hell*:
- snap — самый медленный запуск приложений
- flatpack — запуск приложений практически со скоростью нативных пакетов
- appimage


Как посмотреть версию дистрибутива?

`$ cat /etc/os-release`
