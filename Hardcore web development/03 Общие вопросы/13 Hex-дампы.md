Hex-дамп — это текстовый файл, содержащий символы 16-ричного представления байтов исходного файла, фрагмента памяти и т. п.

`$ xxd my_file.txt` — возвращает файл в виде последовательности байтов
`$ xxd -c 1 -b my_file.txt` — список байтов в одну колонку, в двоичном, а не в 16-ричном виде

Таким образом можно превратить исполняемый файл в текстовый файл:
`$ xxd -p calc.exe calc_dump.txt`

А потом переслать этот текстовый файл по почте, после чего восстановить исполняемый файл обратно из его текстового представления:
`$ xxd -p -r calc_dump.txt calc.exe`

Почтовые системы имеют в своём арсенале огоромное количество способов для определения, является ли файл исполняемым.

Самый простой способ — по заголовку из первых двух байтов, с помощью которого Windows определяет тип файла
4D 5A — DOS-исполняемый файл,
FF D8 — изображение в формате jpeg.

Самый сложные способы  — попытка запустить файл в специальном изолированном контейнере или применение специальной нейронной сети, натренированной на обнаружение потенциально исполняемых файл.
