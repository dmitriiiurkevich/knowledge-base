### Iterable, Iterator, Generator

iterable, iterator, generator – это три абстрактных класса, определённых в пакете collections.abc.

#### Iterable

абстрактный класс collections.abc.Iterable 

содержит абстрактный метод __iter__, возвращающий итератор  

  

или

  

абстрактный класс collections.abc.Sequence

содержит абстрактный метод __getitem__, возвращающий элемент по индексу

#### Iterator

абстрактный класс collections.abc.Iterator

наследует collections.abc.Iterable

содержит абстрактный метод __next__, возвращающий очередной элемент или выбрасывающий исключение StopIteration.

содержит реализованный метод __iter__, возвращающий self, что позволяет использовать Iterator вместо Iterable.

  

NB

Оператор iter может сделать генератор из любого callable объекта:

  

from random import randint

  

def d6():

   return randint(1, 6)

  

for roll in iter(d6, 6):

   print(roll)

  
  
  

#### Generator

абстрактный класс collections.abc.Generator

наследует collections.abc.Iterator

содержит абстрактные методыы send, throw.

содержит реализованные методы __iter__, __next__, и close. 

  

Метод send.

  

Передаёт параметр внутрь генератора. С помощью этого параметра можно управлять поведением генератора.

  

def gen():

   i = 0

   while True:

       x = yield i

       i += x

  

g = gen()

print(next(g))

print(g.send(11))

print(g.send(22))

  

# output:

# 0

# 11

# 33

# 66

  
  

Метод throw.

  

Заставляет генератор выбросить исключение.

  

Путаница в терминологии: [https://docs.python.org/3/glossary.html#term-generator](https://docs.python.org/3/glossary.html#term-generator)

  

generator – согласно документации – это функция, содержащая yield. Результатом запуска такой функции является так называемый generator iterator.

  

При этом формально:

- объект generator имеет тип function,
    
- объект generator iterator имеет тип generator.
    

  

generator expression – выражение вида: (i*i for i in range(10))

combined expression – выражение вида: sum(i*i for i in range(10))

  

При этом формально результатом вычисления generator expression будет объект типа iterator.
