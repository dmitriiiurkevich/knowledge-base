`wait_time` — attribute — a generator — function that yields a delays after each task execution.

`constant(wait_time)` — return a function that always returns a given constant value

`between(min_wait, max_wait)` — return a function that returns a random value from a given range

`constant_paicing(wait_time)` — return a function that tries to equalize the tasks execution time

```python 
class MyUser(User):
    wait_time = constant_pacing(10)
    @task
    def my_task(self):
        time.sleep(random.random())
```

— the task will always be executed once every 10 seconds, no matter the task execution time
— If a task execution exceeds the specified wait_time, the wait will be 0 before starting the next task

`constant_throughput(task_runs_per_second)` — return a function that make the number of task runs per second execution equal to a given value

```python 
class MyUser(User):
    wait_time = constant_throughput(0.1)
    @task
    def my_task(self):
        time.sleep(random.random())
```

— the task will always be executed once every 10 seconds, no matter the task execution time
— If a task execution exceeds the specified wait_time, the wait will be 0 before starting the next task

constant throughput × number of users = expected iterations per second

If you want Locust to run 500 task iterations per second, you could use 
- 5000 users
- `wait_time = constant_throughput(0.1)`

But if the time for the task iteration exceeds 10, the throughput will be less than 500.

Swarm ramp-up +  high *spawn rate/ramp* → you may end up exceeding the target value, because the *wait_time*  is applied _after_ first task execution

The default value:  `wait_time = constant(0)`

It’s also possible to declare your own `wait_time` method directly on your class.

```python 
class MyUser(User):
    last_wait_time = 0

    def wait_time(self):
        self.last_wait_time += 1
        return self.last_wait_time
    
    # ...
```
