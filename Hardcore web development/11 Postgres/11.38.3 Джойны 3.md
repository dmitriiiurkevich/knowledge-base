*SEMI JOIN* — это тип соединения двух таблиц, который возвращает только записи из первой таблицы. При этом из всех строк первой таблицы выбираются только те строки, для которых существует соответствующая им строки во второй таблице.


```sql
select b.name
from book b
where exists (
    select from author a where a.author_id = b.author_id
);
```

В этом запросе вообще нет ключевого слова *JOIN*, вместо которого используется ключевое слово *EXISTS*.

Этот же запрос можно переписать с помощью ключевого слова  *IN*

```sql
select b.name
from book b
where b.author_id in (
    select author_id from author a
);
```

Существует мнение, что версия запроса, использующая слова *JOIN*, является более читаемой.

Важный вопрос
Что вернёт следующий запрос, при условии, что существуют все используемые в запросе таблицы и атрибуты?

```sql
select * from a
where exists (
   select * from b where b.some_field = 'value'
);
```

В случае, если подзапрос вернёт хоть одну строку, то `exists` вернёт `true`, и тогда внешний запрос вернёт все строки таблицы `a`. В этом случае, рассматриваемый запрос будет идентичен любому из следующих запросов:

```sql
select * from a where exists (select true)
```

```sql
select * from a where true;
```

В случае, если подзапрос не вернёт ни одной строки, то `exists` вернёт `false`, и тогда весь запрос тоже не вернёт ничего. В этом случае, рассматриваемый запрос будет идентичен любому из следующих запросов:

```sql
select * from a where exists (select 1 where 1 = 0);
```

```sql
select * from a where false;
```


*ANTI JOIN* — это противоположность *SEMI JOIN* — тип соединения двух таблиц, который также возвращает только записи из первой таблицы. Но из всех строк первой таблицы выбираются только те строки, для которых НЕ существует соответствующая им строки во второй таблице.

```sql
select b.name
from book b
where not exists (
    select from author a where a.author_id = b.author_id
);
```

Важное замечание
Запрос с конструкцией *SEMI JOIN* можно переписать, используя ключевое слово *IN* (это может негативно сказаться на скорости выполнения запроса, но результат выполнения запроса останется тем же).

А вот запрос с конструкцией *ANTI JOIN* не всегда можно просто переписать, используя ключевое слово *NOT IN*.

```sql
select b.name
from book b
where b.author_id not in (
    select author_id from author a
);

-- этот запрос не вернёт те строки таблицы book,
-- у которых значение атрибута book.author_id = null,
-- поскольку null не сравним ни с числом, ни с другим null
```
