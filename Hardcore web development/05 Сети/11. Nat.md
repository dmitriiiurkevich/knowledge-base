
*NAT* *(Network Address Translation)* — технология преобразования *IP*-адресов внутренней (локальной) сети в *IP*-адреса внешней сети *(Internet)* в заголовках *IP*-пакетов, разработанная с целью преодоления нехватки свободных *IP*-адресов стандарта *IPv4*.

→ [[12.1. IP-адреса]]

Предположим, что у нас есть внутренняя сеть с четырьмя компьютерами, каждый из которых имеет свой внутренний *IP*-адрес.

И предположим, что эта сеть подключена к интернет или к внешней сети через специальное *NAT*-устройство, которое имеет два различных *IP*-адреса:
1. внутренний *IP*-адрес, 
	- видимый только из внутренней сети,
	- используемый для маршрутизации пакетов во внутренней сети.
2. внешний *IP*-адрес,
	- видимый из сети *Internet* или из внешней сети,
	- используемый для маршрутизации пакетов в сети *Internet* или во внешней сети.

| Internal *LAN*                                                                                                                           |     | *NAT* device                                                                                      |           | The Internet |
| ---------------------------------------------------------------------------------------------------------------------------------------- | :-: | ------------------------------------------------------------------------------------------------- | :-------: | :----------: |
| Host 1 internal ip: 192.168.0.1<br>Host 2 internal ip: 192.168.0.2<br>Host 3 internal ip: 192.168.0.3<br>Host 4 internal ip: 192.168.0.4 |  ↔  | *NAT* internal ip: 192.168.1.1<br>                   ↑      ↓<br>*NAT* external ip: 184.86.48.128 | <br><br>↔ | <br><br>🌐︎  |

При отправке пакета из внутренней сети во внешнюю сеть или в сеть *Internet*, *NAT*-устройство автоматически преобразует внутренний *IP*-адрес отправителя, используемый в заголовке отправленного *IP*-пакета, во внешний *IP*-адрес.

И наоборот, при получении из внешней сети или из сети *Internet* ответного *IP*-пакета, предназначенного для одного из компьютеров внутренней сети, *NAT*-устройство автоматически преобразует внешний *ip*-адрес, используемый в заголовке полученного пакета, во внутренний *IP*-адрес получателя.

# Типы технологии *NAT*

## Source NAT
Обеспечивает трансляцию *IP*-адресов для соединений
- с хостами внешней сети,
- инициируемых хостами внутренней сети.

### Статический *NAT*
Статическое отображение *IP*-адресов один к одному

Необходим пул внешних *IP*-адресов, содержащий отдельный внешний *IP*-адрес, для каждого хоста внутренней сети. Такой подход применяется при инкорпорировании внутренней локальной сети в более объемлющую внешнюю локальную сеть для того, чтобы избежать возможного конфликта *IP*-адресов.

### Динамический *NAT*
Динамическое отображение внутренних *IP*-адресов на группу из нескольких внешних *IP*-адресов. 

Необходим пул из нескольких внешних *IP*-адресов, которые поочерёдно используются различными компьютерами внутренней сети для динамического отображения своих внутренних *IP*-адресов.

### *Masquerading NAT*, *PAT*
Необходим всего один внешний *IP*-адрес. При этом пара 
1. внутренний *ip*-адрес хоста, отправляющего исходящий запрос во внешнюю сеть,
2. порт исходящего запроса
динамически отображается на множество свободных портов *NAT* устройства, образуя новую пару
1. фиксированный внешний *IP*-адрес *NAT*-устройства,
2. слцчайный порт, выбранный из множества свободных портов *NAT* устройства и сохранённый в коммутационной таблице *NAT*-устройства.

И хотя общее количество свободных портов *NAT*-устройства ограничено числом 2¹⁶=65535, их оказывается достаточно для того, чтобы через единственный внешний *IP*-адрес локального *Internet*-провайдера в сеть выходили бы жители небольшого микрорайона.

Эта технология также называется *Port Address Translation* — *PAT* и является наиболее часто применяемым типом технологии *NAT*.

### Алгоритм работы *PAT*

##### Шаг 1
Браузер, запущенный на Host 2 инициирует соединение, отправляя *IP*-пакет, заголовок которого содержит следующие параметры:
* протокол: *TCP*  
* *IP*-адрес назначения: 142.251.1.102 (google.com)
* *TCP*-порт назначения, 443 (https)  
* *IP*-адрес источника, 192.168.0.2 (внутренний *IP*-адрес Host 2)  
* *TCP*-порт источника, 4242

##### Шаг 2
Отправленный на предыдущем шаге *IP*-пакет попадает в *NAT*-устройство. *NAT*-устройство создаёт в своей таблице маршрутизации запись следующего вида:

| $n$ | 192.168.0.2:4242 | 4502 | $timeout(n)$ |
| --- | ---------------- | ---- | ------------ |

- $n$ — номер строки
- 192.168.0.2:4242 — *IP*-адрес и *TCP*-порт источника соединения
- 4502 — случайно выбранный свободный порт *NAT*-устройства
- $timeout(n)$ — настраиваемый таймаут, по истечении которого данная запись будет удалена из таблицы маршрутизации 

##### Шаг 3
*NAT*-устройство модифицирует заголовок  *IP*-пакета, заменяя в нём *IP*-адрес и *TCP*-порт:
* протокол: *TCP*  
* *IP*-адрес назначения: 142.251.1.102 (google.com)
* *TCP*-порт назначения, 443 (https)  
* *IP*-адрес источника, 184.86.48.128 (внешний *IP*-адрес *NAT*-устройства)  
* *TCP*-порт источника, 4502

##### Шаг 4
*NAT*-устройство отправит во внешнюю сеть *IP*-пакет с модифицированным заголовком.

##### Шаг 5
Когда *NAT*-устройство получит из внешней сети ответный пакет, то произойдет обратная трансляция, и адрес 184.86.48.128:4502 в заголовке ответного пакета будет заменён на 192.168.0.2:4242. 

##### Шаг 6
*NAT*-устройство отправит во внутреннюю сеть ответный *IP*-пакет с модифицированным заголовком.

## Destination NAT
Обеспечивает трансляцию *IP*-адресов для соединений
- с хостами внутренней сети,
- инициируемых хостами внешней сети.

Представляет собой статическое отображение некоторых портов в таблице маршрутизации *NAT*-устройства:
- порт 80 *NAT*-устройства → порт 80 внутреннего *Web*-сервера
- порт 25 *NAT*-устройства → порт 25 внутреннего *SMTP*-сервера
- порт 21 *NAT*-устройства → порт 25 внутреннего *FTP*-сервера

## Преимущества технологии *NAT*
- Позволяет преодолеть нехватку адресов *IPv4*.
- Легко развернуть и использовать, поскольку для этого необходимо всего одно довольно простое устройство.
- Скрывает структуру внутренней сети от злоумышленников из внешнего мира — уровня безопасности выше.

## Недостатки технологии *NAT*
- Технология *NAT* нарушает фундаментальный принцип построения *IP*-сетей: любой компьютер должен иметь возможность соединиться с любым другим компьютером, поскольку не позволяет компьютерам внешней сети напрямую подключиться к компьютерам внутренней сети.
- Некоторые протоколы, такие как *FTP*, расчитанные на возможность прямого соединения между хостами, плохо работают через *NAT*.
- Однако, изменеия в таблице маршрутизации *NAT* могут вызывать проблемы в работе даже тех протоколов, которые не устанавлвают прямого соединения между хостами.
- Нет единого стандарта *NAT*.

По умолчанию через *PAT* пройдёт только *TCP*, *UDP* и *ICMP* трафик. Всё остальное будет отброшено. Существует возможность пропустить через *PAT* и некоторые другие протоколы, но для этого требуется глубокий и ресурсоемкий анализ сессии. Такой функционал многие производители сетевого оборудования называют *application layer gateway* — *ALG*.  

## Решение проблем, связанных с технологией *NAT*
Для некоторых *IP*-адресов внутренней сети можно настроить статическое отображение на небольшой пулл внешних *IP*-адресов.  

Для некоторых портов можно также настроить статическое отображение — *Destination NAT*. Для этого никаких дополнительных внешних *IP*-адресов не потребуется.

Существует технология *NAT Traversal* (стандарт *RFC 3489*), позволяющая устанавливать прямое соединения с компьютерами во внутренней сети, некоторые варианты которой часто используются различными *VoIP* приложениями.
