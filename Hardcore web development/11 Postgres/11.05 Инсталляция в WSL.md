## Удаляем старую версию *Postgres*

```
$ sudo apt-get --purge remove postgresql postgresql-*
```


Получить список пакетов, содержащих в своём названии слово *postgres*, и в случае, если он оказался не пустым, удалить эти пакеты руками

```
$ dpkg -l | grep postgres
```

Удалить директории, в которые была установлена старая версия

```
$ sudo rm -rf /var/lib/postgresql/
$ sudo rm -rf /var/log/postgresql/
$ sudo rm -rf /etc/postgresql/
```

Удалить пользователя *postgress*

```
$ sudo deluser postgres
```

## Устанавливаем актуальную версию *Postgres*

Инструкция по установке актуальной 16-й версии *Postgres* на *Linux* (подходит для *WSL2*):
https://dev.to/johndotowl/postgresql-16-installation-on-ubuntu-2204-51ia
\
Устанавливаем пароль пользователя *postgress*

```
$ sudo passwd postgres
```

Подключаемся к *Postgres* локально

```
$ su postgres
Password: ...
```

Проверяем актуальность установленной версии

```
$ psql
...

# SELECT version();
...

q
```

``` shell

```


## Создаём базу данных

```sql
# CREATE ROLE rroom WITH LOGIN PASSWORD 'rroom';
CREATE ROLE

# CREATE DATABASE rroom_db
    WITH
    TEMPLATE=template0
    ENCODING='UTF8'
    LC_COLLATE='ru_RU.UTF-8'
    LC_CTYPE='ru_RU.UTF-8'
    owner rroom;
CREATE DATABASE
```


```sql
# SHOW data_directory;
       data_directory
-----------------------------
 /var/lib/postgresql/16/main
(1 row)
```


```sql
# SHOW config_file;
               config_file
-----------------------------------------
 /etc/postgresql/16/main/postgresql.conf
(1 row)
```

Дополнительно в этом конфигурационном файле можно раскомментировать параметр

```
#------------------------------------------------------------------------------
# CONNECTIONS AND AUTHENTICATION
#------------------------------------------------------------------------------
# - Connection Settings -
# listen_addresses = 'localhost'            # what IP address(es) to listen on;
#------------------------------------------------------------------------------
```

для WSL2: `listen_addresses = 'localhost'`
для виртуальной машины: `listen_addresses = '*'`

После модификации конфигурационного файла, необходимо перезагрузить *Postgres*

```
$ sudo service postgresql stop
$ sudo service postgresql start
```
