
### Коротко

Компилируемые языки — *compiled languages* — C, C++, Go, Rust — преобразуют программу в исполняемый файл с помощью компилятора, после чего, для запуска программы на исполнение, необходимо будет запустить этот самый исполняемый файл.

После запуска исполняемого файла операционная система создаст новый процесс, в _text segment_ разделе виртуальной памяти которого, будет загружен код исполняемого файла.

Интерпретируемые языки — interpreted languages — Python, JS, Java — не преобразуют программу в исполняемый код, поэтому для того, чтобы запустить на исполнение неоткомпилированную программу, из командной строки запускается интерпретатор соответствующего языка, которому одним из параметров вызова передаётся путь файлу, содержащему текст запускаемой программы.

Операционная система создаёт процесс, в _text segment_ разделе виртуальной памяти которого, загружается код интерпретатора. А текст запускаемой программы загружается в _data segment_ соответствующего процесса, как одна из глобальных переменных интерпретатора, поскольку _data segment_ в случае интерпретируемых языков будет содержать глобальные переменные не исполняемой программы, а интерпретатора языка.

Переменные же, объявленные в вашей интерпретируемой программе, не попадают в _data segment_, а находятся в _heap_ процесса интерпретатора как часть среды выполнения языка.

### Преимущества и недостатки

Компилируемые языки:
- обычно обеспечивают более высокую производительность
- обычно более платформозависимы
- допускают прямое взаимодействие с ОС
	- низкоуровневый контроль над ОС и аппаратной частью даёт массу преимуществ
	- усложняет синтаксис
	- проще отстрелить себе ногу

Интерпретируемые языки:
- обычно обеспечивают более низкую производительность
- обычно менее платформозависимы
- за взаимодействие с ОС отвечает виртуальная машина
	- синтаксис проще
	- разработка и отладка проще — до тех пор, пока кому-нибудь не потребуется глубокое профилирование, ибо он будет профилировать не столько процесс своей программы, сколько процесс интерпретатора
	- труднее отстрелить себе ногу

### Подробно

Первоначальное представление любой программы, написанной на любом языке программирования, — это исходный текст. 

В процессе выполнения программы этот текст последовательно проходит через определённое количество промежуточных представлений. Специальные преобразователи последовательно преобразуют выполняемую программу, переводя её из предыдущего промежуточного представления в следующее.

Последнее представление выполняемой программы – это всегда набор команд машинного кода.  

Машинный код всегда платформо-зависим. Машинный код всегда интерпретируется, хотя интерпретация машинного кода может происходить непосредственно на аппаратном уровне.

Некоторые промежуточные представления могут сохраняться в виде файлов. 

Некоторые промежуточные представления существуют только в памяти компилятора и только непосредственно в процессе работы компилятора (хотя, если это потребуется, могут быть выгружены из памяти с помощью специальных средств).

Например, бинарные файлы при выполнении *C* программы или класс-файлы при выполнении *Java* программы сохраняются на диске.

Файлы *.pyc*, содержащие байт-код *PVM* — виртуальной машины *Python* — также могут быть сохранены на диске, хотя обычно этого не происходит.

А при выполнении в браузере программы, написанной на языке *JavaScript*, никакие промежуточные состояния не сохраняются в виде файлов на диске.

Финальные представления программы формируются уже непосредственно внутри процессора. Например, такое состояние может быть результатом преобразования команд, считанных из бинарного файла (предварительно загруженного в оперативную память), в команды микрокода процессора.

Выполнение программы завершается интерпретацией получившейся в результате последовательности команд микрокода процессора.

*Компиляцией* называется процесс преобразования внутренних промежуточных состояний программы внутри памяти виртуальной машины. 

В этом смысле стадию компиляции при своём выполнении проходят программы, написанные не только на C, но и на Java, на Python или на JS – языках, традиционно называемых интерпретируемыми.
